hexo-GitHub方法搭建个人博客

发表于2023-07-06|更新于2024-06-19|前端
|阅读量:

一些杂谈

之前我自己搭建过个人博客，当时我记得我通过购买腾讯云服务器，购买域名，备案…走了很多弯路，当时通过wordpress，套用他人的theme等一系列操作，好不容易搞出来一个稍微像样的，后期疏于维护并且服务器到期了，就采用了hexo建立博客，并且在github page平台上托管方法，搭建了这个小破blog。

总的来说，步骤比较简单，主要是很多零碎细杂的问题不能面面俱到。

在这之前，我们需要下载markdown，这不得不说是个好东西！Typora — a markdown editor, markdown reader.

搭建教程

安装git

Q：为什么要安装git？

A：我们需要把本地文件上传到github上面去，我们就需要用到git，git可以有效快速的处理从很小到很大的项目版本管理。

首先，我们要到git官网上下载.exe文件，网址：Git (git-scm.com)

安装node.js

Hexo是基于node.js编写的，我们需要node.js以及里面的npm工具。

建立直接去[node.js官网](Node.js (nodejs.org))下载长期稳定版，解压到指定的文件夹即可

检测是否安装好：打开cmd，输入命令node -v以及npm -v如果出现版本号，那么就表示安装成功。

添加国内镜像源：

如果没有科学上网的话，我们用阿里的国内镜像加速

安装hexo

前面两个都安装好之后，我们就可以安装hexo了，我们可以创建一个文件夹blog，存放自己的博客文件，如何cd到这个文件夹下。

打开命令行窗口（win+r）

比如我的博客就存放在F:\blog目录下，要cd到这个目录就cd /d F:blog就行了。

定位到博客根目录下，我们在cmd中输入npm install -g hexo-cli安装Hexo

安装完后输入hexo -v 验证是否安装成功

接着我们来初始化一些hexo，同样的，进入根目录，输入hexo init初始化文件夹，

然后输入npm install安装必备的组件，新建完成后，指定hexo文件夹目录下有node_modules public 等等文件夹，其中较为重要的是 _config.yml这是博客的配置文件

到这步，本地网站的配置也干好了，输入hexo g生成静态网页，然后输入hexo s打开本地服务器，localhost:4000，你就能看到一个简单的界面了，在命令行中按ctrl+c关闭本地服务器。

注册github账号并创建个人仓库

接下来就去注册一个github账号，用来存放我们的网站。大多数小伙伴应该都有了吧。

打开https://github.com/，新建一个项目仓库`New repository`，如下所示

必须严格按照这个格式来，如我的就应该在空格内填写``littleforest6.github.io`因为只有这样，后面才会被github page识别。

然后创建SSH,一路回车

ssh，简单来讲，就是一个秘钥，其中，id_rsa是你这台电脑的私人秘钥，不能给别人看的，id_rsa.pub是公共秘钥，可以随便给别人看。把这个公钥放在GitHub上，这样当你链接GitHub自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过git上传你的文件到GitHub上。(有没有一点像算法中的密码算法)

恭喜你已经完成第五步了，欸嘿。

修改theme

从github上找到喜欢的theme，每个hexo的theme下面都可以找到相关教程，在博客的根目录下，跟着每个theme不同的教程来更换博客主题。

一般只需要修改_config.yml文件或者不同主题的yml文件就行了。

2024.06.19日问题

hexo d时候的spawn failed问题

F:/blog/.deploy_git权限故障

解决方法：git bash 中 git config --global --add safe.directory F:/blog/.deploy_git

随后再次hexo d即可
